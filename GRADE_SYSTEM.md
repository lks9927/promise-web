# ⭐ 10년의 약속 - 파트너 등급 시스템 명세서

> **이 문서는 파트너(딜러/팀장)의 등급 관리, 승급 조건, 취소 보호 프로세스를 정의합니다.**
> **모든 AI 및 개발자는 관련 코드 수정 전 반드시 이 문서를 먼저 읽어야 합니다.**

## 1. 등급 구조

### 등급 경로
```
C (신입) → B (일반) → A (우수) → 마스터 승격 대상
```

### 등급별 의미

| 등급 | 이름 | 설명 |
|:----:|------|------|
| **C** | 신입 | 신규 등록 시 기본 등급 |
| **B** | 일반 | 일정 실적을 쌓은 파트너 |
| **A** | 우수 | 품질+실적 모두 검증된 파트너 |
| **Master/S** | 마스터 | 하위 파트너를 관리하는 리더 (관리자 승인 필요) |

### 마스터와 일반 등급의 핵심 차이

| 구분 | 일반 (A/B/C) | 마스터 (Master/S) |
|------|:-----------:|:----------------:|
| 하위 파트너 보유 | ❌ 불가 | ✅ 가능 |
| 관리 커미션 수령 | ❌ 없음 | ✅ 발생 |
| 직접 업무 커미션 | ✅ 동일 금액 | ✅ 동일 금액 |
| 등급 변경 방식 | 자동 승급 | 관리자 수동 승인 |

> **핵심**: 정산 트리거는 등급(grade)을 보지 않고 `master_id` 연결관계만 본다.
> 등급은 UI 표시 + 승급 관리용이며, 돈의 흐름은 `master_id`가 결정한다.

## 2. 포인트 시스템

### 2.1 포인트 획득 규칙

| 항목 | 조건 | 포인트 | 비고 |
|------|------|:------:|------|
| 마스터 평가 ★★★★★ | 별 5개 | **+2점** | 메인 평가 |
| 마스터 평가 ★★★★ | 별 4개 | **+1점** | |
| 마스터 평가 ★★★ | 별 3개 | **0점** | 보통 |
| 마스터 평가 ★★ | 별 2개 | **-1점** | 경고 |
| 마스터 평가 ★ | 별 1개 | **-2점** | 심각 |
| 고객 만족도 ★★★★★ | 별 5개 | **+2점** | 보너스 |
| 고객 만족도 ★★★★ | 별 4개 | **+1점** | 보너스 |
| 장례 1건 완료 | 정산 완료 기준 | **+1점** | 기본 완료 보상 |

### 2.2 패널티 규칙

| 상황 | 패널티 | 사유 |
|------|:------:|------|
| 전화 상담 후 취소 | **0점** | 탐색 단계, 취소 자연스러움 |
| 방문 상담 후 취소 | **-1점** | 시간/비용 투입했으나 계약 전 |
| 계약 후 취소 (진행 중 해지) | **-2점** | 심각한 문제, 강한 패널티 |

> **설계 의도**: 상담은 부담 없이 많이 받고, 계약은 확실하게 하도록 유도

### 2.3 승급 조건 (포인트 + 실적 + 기간 복합)

| 승급 | 누적 포인트 | 최소 완료 건수 | 최소 활동 기간 | 방식 |
|------|:---------:|:----------:|:----------:|------|
| **C → B** | 20점 이상 | 5건 이상 | 1개월 | 자동 승급 |
| **B → A** | 50점 이상 | 15건 이상 | 3개월 | 자동 승급 |
| **A → 마스터 대상** | 100점 이상 | 30건 이상 | 6개월 | ⚠️ 관리자 승인 필요 |

> **3가지 조건을 모두 충족해야 승급**
> - 포인트만 높고 건수 부족 → 안 됨
> - 건수만 많고 평점(포인트) 낮음 → 안 됨
> - 둘 다 충족해도 기간 미달 → 안 됨

## 3. 취소 보호 프로세스

### 3.1 문제 상황

> 딜러가 영업해서 가져온 건인데, 팀장이 취소 치고 상주와 개인 계약하면?
> → **딜러 커미션 탈취 + 본사 수수료 탈취 = 사기**

### 3.2 취소 프로세스 (필수 절차)

```
팀장이 "취소" 버튼 클릭
       ↓
┌──────────────────────┐
│ ① 취소 사유 필수 입력   │ ← 빈칸이면 취소 불가
│   (텍스트 기록 = 증거)  │
└──────────┬───────────┘
           ↓
┌──────────────────────┐
│ ② 접수 딜러에게 즉시 알림│ ← 푸시 + 앱 내 알림
│   "○○○ 건이 취소됨"    │
│   사유: "고객 변심"     │
└──────────┬───────────┘
           ↓
┌──────────────────────┐
│ ③ 딜러 확인/이의 제기   │
│   - "확인" → 정상 종료  │
│   - "이의 제기" → 본사  │
│   - 3일 미확인 → 자동   │
│     에스컬레이션        │
└──────────┬───────────┘
           ↓
┌──────────────────────┐
│ ④ 본사 취소 모니터링    │
│   - 취소율 높은 팀장 플래그│
│   - 취소 이력 영구 보관  │
└──────────────────────┘
```

### 3.3 부정행위 방지 장치

| 방지 수단 | 설명 |
|----------|------|
| **취소 사유 필수** | 사유 미입력 시 취소 버튼 비활성화 |
| **딜러 즉시 알림** | 내 건이 취소되면 실시간 알림 |
| **딜러 확인/이의제기** | 딜러가 이상하면 본사에 통보 |
| **재접수 차단** | 취소된 고객 번호로 일정 기간 재접수 불가 |
| **취소 이력 영구 보관** | 누가, 언제, 왜 취소했는지 기록 |
| **취소율 모니터링** | 특정 팀장 취소율 비정상 시 자동 경고 |

### 3.4 취소 보고 체계

| 이해관계자 | 알림 내용 | 타이밍 |
|----------|---------|--------|
| **접수 딜러** | 건 취소 알림 + 사유 | 즉시 |
| **마스터 딜러** (있으면) | 하위 딜러 건 취소 | 즉시 |
| **마스터 팀장** (있으면) | 하위 팀장 건 취소 | 즉시 |
| **본사 관리자** | 취소 건 대시보드 집계 | 실시간 |
