# 🧪 10년의 약속 — QA 버그 테스트 체크리스트

> 작성: 2026-04-16  
> **목적**: 실제 사용자처럼 화면을 클릭하며 점검하는 체크리스트  
> **규칙**: 각 항목을 직접 실행하고, 이상 있으면 ❌ 표시 + 증상 메모

## 🔑 1. 로그인 / 인증

### 1-1. 정상 로그인
- [ ] 관리자 계정으로 로그인 → 관리자 대시보드 진입 확인
- [ ] 마스터팀장 계정 → 마스터 대시보드 진입 확인
- [ ] 팀장 계정 → 팀장 대시보드 진입 확인
- [ ] 마스터딜러 계정 → 딜러 대시보드 진입 확인
- [ ] 딜러 계정 → 딜러 대시보드 진입 확인

### 1-2. 비정상 로그인
- [ ] 잘못된 비밀번호 → 에러 메시지 정상 출력? (무한 로딩 아닌지)
- [ ] 빈 아이디/비밀번호 → 적절한 경고 표시?
- [ ] 비밀번호 1자리만 입력 → 최소 6자리 경고?
- [ ] 존재하지 않는 계정 → 에러 메시지?
- [ ] **로그인 후 뒤로가기** → 로그인 화면으로 다시 가나? 대시보드 유지?

### 1-3. 로그아웃
- [ ] 로그아웃 클릭 → 로그인 화면으로 복귀?
- [ ] 로그아웃 후 **직접 URL 입력**으로 대시보드 접근 시도 → 차단되나?

### 1-4. 세션/새로고침
- [ ] 로그인 상태에서 **F5 새로고침** → 로그인 유지?
- [ ] 브라우저 탭 닫고 다시 열기 → 로그인 유지?

## 📋 2. 장례 접수 (고객/랜딩페이지)

### 2-1. 랜딩페이지 (Home)
- [ ] 페이지 정상 로딩? (빈 화면 아닌지)
- [ ] 모바일 화면에서 깨지지 않는지? (크롬 DevTools → 모바일 모드)
- [ ] 스크롤 정상 동작?
- [ ] 상품 선택 버튼 클릭 → 접수 폼 이동?

### 2-2. 접수 폼
- [ ] 이름 빈값으로 제출 → 에러 메시지?
- [ ] 연락처 빈값으로 제출 → 에러 메시지?
- [ ] 연락처에 문자 입력 → 숫자만 허용?
- [ ] 연락처 짧게 입력 (예: 010-1234) → 유효성 경고?
- [ ] 상품 선택 없이 제출 → 경고?
- [ ] **모든 값 정상 입력 후 제출** → 성공 메시지 + DB 기록?
- [ ] 제출 버튼 **연타** → 중복 접수 방지?
- [ ] 접수 후 고객 상태 확인 페이지 진입 가능?

## 👨‍💼 3. 관리자 대시보드

### 3-1. 접수 현황 (CaseTab)
- [ ] 접수 목록 정상 로딩? (로딩 무한 아닌지)
- [ ] 새 접수 건 실시간 반영?
- [ ] 접수 상세 클릭 → 상세 모달 정상 표시?
- [ ] 상태 변경 (접수→진행→완료) → 저장 + 화면 반영?
- [ ] **건이 0개일 때** 빈 상태 → 적절한 안내 메시지?

### 3-2. 파트너 관리 (PartnerTab)
- [ ] 전체 파트너 목록 로딩?
- [ ] 유저 역할별 필터링 정상?
- [ ] 유저 상태 변경 (승인/대기/차단) → 저장?
- [ ] 마스터-딜러 관계 표시 정상?
- [ ] **하위 인원 0명인 마스터** 표시 정상?

### 3-3. 수수료 설정 (CommissionSettings)
- [ ] 수수료 테이블 정상 로딩?
- [ ] 금액 입력 → **소숫점 없이 정수만** 표시? ⭐
- [ ] 비율(%) 토글 → 자동 계산 정상?
- [ ] 비율 모드에서 시뮬레이션 → **소숫점 없는 정수** 표시? ⭐
- [ ] 비율 33.33% 같은 값 입력 → 계산 결과 정수? ⭐
- [ ] 장례수수료(base_margin)를 0원으로 → 0으로 나누기 에러 없는지?
- [ ] 장례수수료를 음수로 → 방어 처리?
- [ ] 시뮬레이션 6개 케이스 → 모든 영업이익이 정수?
- [ ] **저장 버튼** 클릭 → 성공 토스트 + DB 반영?
- [ ] 저장 후 새로고침 → 값 유지?
- [ ] 상품 추가 / 삭제 → 정상 동작?
- [ ] 상품명 수정 → 저장 반영?

### 3-4. 정산 관리 (SettlementManager)
- [ ] 정산 목록 로딩?
- [ ] 금액 표시에 **소숫점 없는지**? ⭐
- [ ] 정산 상태 변경 → 저장?
- [ ] 합계 금액 계산 정상?

### 3-5. 쿠폰 발급 (CouponPanel)
- [ ] 쿠폰 목록 로딩?
- [ ] 쿠폰 생성 → 저장?
- [ ] 중복 쿠폰 코드 → 에러 처리?

### 3-6. 설정 (SettingsPanel)
- [ ] 랜딩 이미지 업로드 정상?
- [ ] 설정 저장 → 반영?

### 3-7. 메시지 (AdminMessageTab)
- [ ] 메시지 발송 → 수신자에게 도착?
- [ ] 빈 내용으로 발송 시도 → 차단?

- [ ] 사진 없이 보고 → 정상 동작?

### 4-3. 정산/수입 확인
- [ ] 팀장 정산 내역 표시?

## 💰 9. 금액 관련 통합 점검 ⭐⭐⭐

> **원(₩)은 절대 소숫점이 나오면 안 됩니다.**

### 모든 화면에서 금액 확인:
- [ ] 관리자 수수료 설정 → 정수만?
- [ ] 관리자 시뮬레이션 → 정수만?
- [ ] 관리자 정산 관리 → 정수만?
- [ ] 팀장 정산 내역 → 정수만?
- [ ] 딜러 정산 내역 → 정수만?
- [ ] 쿠폰 금액 → 정수만?

### 금액 계산 정합성:
- [ ] 장례수수료 - (모든 지급) = 영업이익 → 정확?
- [ ] 비율(%) 모드 → 금액(원) 모드 전환 시 값 일관?
- [ ] 극단값: 장례수수료 1원 → 에러 없이 동작?
- [ ] 극단값: 장례수수료 9,999,999원 → 화면 깨짐 없이 표시?

## 🔄 10. 전체 흐름 시나리오 테스트

### 시나리오 A: 일반딜러 영업 → 일반팀장 진행
1. [ ] 딜러 로그인 → 고객 등록/접수
2. [ ] 관리자 확인 → 접수 건 표시됨
3. [ ] 순차배정 → 팀장에게 배정됨
4. [ ] 팀장 수락 → 진행 시작
5. [ ] 팀장 진행 보고 (사진 + 상태 업데이트)
6. [ ] 장례 완료 처리
7. [ ] 정산 내역 생성 확인 (딜러 20만, 마스터딜러 5만, 고객 10만)
8. [ ] 각 유저 대시보드에서 정산 금액 확인

### 시나리오 B: 본사 직접 접수 → 마스터팀장 진행
1. [ ] 고객이 홈페이지에서 직접 접수
2. [ ] 관리자만 확인 가능 (딜러 쪽 비공개)
3. [ ] 마스터팀장에게 배정 → 수락
4. [ ] 진행 → 완료
5. [ ] 고객 10만원 정산만 발생 (딜러/마스터딜러 없음)
6. [ ] 영업이익 900,000원 확인

### 시나리오 C: 팀장 패스 → 다음 팀장 배정
1. [ ] 1순위 팀장에게 배정
2. [ ] 팀장 패스 클릭
3. [ ] 2순위 팀장에게 자동 이관
4. [ ] 3번 연속 패스/미응답 → 관리자 긴급 알림
