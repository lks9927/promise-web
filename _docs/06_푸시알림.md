# 5단계: 실시간 알림 (PWA + 카카오톡)

> 난이도: 🟡 보통 | 소요: 1~2일 | **핵심 기능**

## 목표
핸드폰 잠금화면에서도 장례 접수/상태 변경 실시간 알림

## 사전 준비 (직접 해야 할 것)

### Firebase 설정
1. https://console.firebase.google.com 접속
2. 프로젝트 생성 (이름: "10년의약속")
3. 웹 앱 추가 → Firebase config값 복사
4. Cloud Messaging → 서버 키 복사
5. `.env.local`에 추가:
```
VITE_FIREBASE_API_KEY=...
VITE_FIREBASE_AUTH_DOMAIN=...
VITE_FIREBASE_PROJECT_ID=...
VITE_FIREBASE_MESSAGING_SENDER_ID=...
VITE_FIREBASE_APP_ID=...
```

### 카카오 비즈니스 (아이폰용)
1. https://business.kakao.com 접속
2. 사업자 등록 (사업자 등록증 필요)
3. 비즈니스 채널 개설
4. 알림톡 템플릿 신청 (승인 1~3일 소요)

필요 템플릿:
```
[긴급 접수 알림]
안녕하세요, 10년의 약속입니다.
새로운 장례 접수가 등록되었습니다.

■ 접수 정보
- 장소: #{location}
- 고객: #{customer_name}
- 상품: #{package_name}
- 접수 시각: #{created_at}

관리자 대시보드에서 확인해주세요.
```

## AI 작업 순서

### 5-1. PWA 기본 설정

```
promise-web을 PWA(Progressive Web App)로 만들어줘.

필요 파일:
1. public/manifest.json — 앱 이름, 아이콘, 색상
   - name: "10년의 약속"
   - short_name: "10년의약속"
   - theme_color: "#4F46E5"
   - display: "standalone"
2. public/sw.js — Service Worker (오프라인 캐시 + 푸시 수신)
3. index.html에 manifest 링크 + SW 등록 코드 추가
4. public/icons/ 폴더에 192x192, 512x512 아이콘 필요 (placeholder로 생성)

결과: 모바일 크롬에서 "홈 화면에 추가" 가능
```

### 5-2. Firebase FCM 연동 (안드로이드)

```
Firebase Cloud Messaging(FCM)으로 푸시 알림을 구현해줘.

1. npm install firebase
2. src/lib/firebase.js 생성 — Firebase 초기화 + messaging
3. Service Worker에서 백그라운드 메시지 수신 처리
4. 로그인 시 FCM 토큰 발급 → profiles 테이블에 fcm_token 컬럼 저장
5. 알림 허용 요청 UI 추가 (최초 1회)

SQL (Supabase에서 실행):
-- ✅ 이미 실행 완료 (2026-04-12)
ALTER TABLE profiles ADD COLUMN fcm_token TEXT;
ALTER TABLE profiles ADD COLUMN os_type TEXT; -- 'android' | 'ios' | 'web'
```

### 5-3. 기기 판별 + 분기 처리

```
로그인 시 안드로이드/아이폰을 자동 판별해서 profiles 테이블에 저장해줘.

1. User-Agent로 기기 판별:
   - /Android/i → 'android'
   - /iPhone|iPad/i → 'ios'
   - 그 외 → 'desktop'
2. 로그인 성공 시 profiles.os_type 업데이트
3. FCM 토큰은 안드로이드/데스크톱만 발급
```

### 5-4. Supabase Edge Function — 알림 발송

```
Supabase Edge Function으로 알림 발송 백엔드를 만들어줘.

파일: supabase/functions/send-alert/index.ts  ← ✅ 생성 완료

로직:
1. funeral_cases INSERT 트리거 → Edge Function 호출
2. 관리자/팀장 profiles 조회 → os_type 확인
3. 분기:
   - os_type = 'android' → FCM HTTP v1 API로 푸시
   - os_type = 'ios' → 카카오 알림톡 API 호출
   - os_type = 'desktop' → FCM 웹 푸시

Supabase Webhook 설정:
- 테이블: funeral_cases
- 이벤트: INSERT
- 대상: Edge Function URL
```

### 5-5. 카카오 알림톡 연동 (아이폰)

```
카카오 알림톡 API를 연동해줘.

1. Edge Function 내에서 카카오 알림톡 발송
2. 필요 환경변수 (Supabase Secrets):
   - SOLAPI_API_KEY (솔라피에서 발급)
   - SOLAPI_API_SECRET (솔라피에서 발급)
3. 발송 대상: os_type이 'ios'인 사용자의 phone 번호
4. 발송 실패 시 → notifications 테이블에 fallback 저장
```

## ⚠️ 주의사항
- 카카오 알림톡은 **사업자 등록 필수** (개인 불가)
- 알림톡 비용: 건당 8~10원
- FCM은 **무료** (무제한)
- iOS Safari는 PWA 푸시가 iOS 16.4+ & 홈화면 추가 시에만 작동
- 그래서 아이폰은 카카오톡으로 대체하는 전략
