# 4.5단계: TimelineView UI 개선

> 난이도: 🟢 쉬움 | 소요: 반나절

## 개선 사항 4가지

### 1. 완료된 장례건 열람 기능
**현재**: 장례 완료(completed) 시 목록에서 사라져 다시 볼 수 없음
**목표**: "지난 장례" 탭 추가 → 완료 건도 열람 가능 (읽기 전용)

### 2. 완료 표시 방식 변경
**현재**: 완료된 단계의 텍스트에 ~~취소선~~이 그어짐 (알아보기 어려움)
**목표**: 텍스트는 그대로 두고, 왼쪽에 ✅ 체크 아이콘 표시

### 3. 사진을 목록 옆에 바로 표시
**현재**: 사진이 텍스트 아래에 숨겨져 있음 (클릭해야 보임)
**목표**: 단계명 오른쪽에 썸네일 사진이 바로 보임

### 4. 스크롤 한 번에 전체 확인
**현재**: 각 단계를 눌러야 사진 확인
**목표**: 마우스 휠로 스크롤하면서 26단계 + 사진 한눈에 확인

## AI 지시 (그대로 복사해서 보내기)

```
src/components/common/TimelineView.jsx를 아래 요구사항대로 개선해줘.

대상 파일: src/components/common/TimelineView.jsx

1. 완료 표시 변경:
   - 현재 완료된 단계 텍스트에 취소선이 있으면 제거
   - 대신 단계명 왼쪽에 초록색 ✅ 체크 아이콘 표시
   - 미완료 단계는 회색 빈 원(○) 표시 (현재와 동일)

2. 사진을 목록 옆에 표시:
   - 각 단계를 가로 레이아웃(flex-row)으로 변경
   - 왼쪽: 체크아이콘 + 단계명 + 작성 시간
   - 오른쪽: 썸네일 사진 (48x48 또는 64x64, 둥근 모서리)
   - 사진이 없는 단계는 회색 카메라 아이콘 표시
   - 썸네일 클릭 시 기존 Lightbox로 크게 보기

3. 스크롤 최적화:
   - 26단계가 세로로 컴팩트하게 나열 (한 단계당 높이 약 60~80px)
   - 마우스 휠/터치 스크롤로 전체를 빠르게 훑을 수 있게
   - 그룹 헤더(상담, 1일차, 2일차, 3일차)는 sticky로 고정

4. 내용(content) 메모가 있으면:
   - 단계명 아래에 작은 글씨로 한 줄 표시 (긴 내용은 말줄임...)
   - 클릭하면 전체 내용 펼치기/접기 토글

주의:
- 기존 useRealtimeSubscription 구독 코드는 유지
- Lightbox(이미지 크게 보기) 기능 유지
- 모바일에서도 잘 보이게 반응형 유지
```

## 완료 건 열람 — 별도 AI 지시

```
TeamLeaderDashboard.jsx에서 완료된 장례 건을 볼 수 있게 해줘.

변경:
1. 현재 myCases는 ['assigned','consulting','in_progress','team_settling','hq_check','completed'] 상태를 조회하는데,
   completed 건이 목록에서 빠르게 사라지는 문제가 있음
2. "내 현황" 탭 안에 서브 토글 추가: [진행 중 | 완료된 장례]
3. "완료된 장례" 선택 시:
   - status가 'completed'인 본인 배정 건 표시
   - 읽기 전용 (상태 변경 버튼 숨김)
   - TimelineView는 그대로 볼 수 있게
4. 마스터 팀장은 하위 팀장의 완료 건도 열람 가능 (현재 myCases 조회 로직 유지)
```
