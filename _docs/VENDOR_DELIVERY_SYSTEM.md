# 외주업체(Vendor) 및 배송기사 시스템 연동 백서

## 1. 개요

* 프로젝트명: 10년의 약속 (`promise-web`)
* 목적: 팀장이 장례 접수 건에 대해 외주업체(장례용품, 입관꽃, 화환)에게 상품을 발주하고, 외주업체가 기사를 통해 해당 병원/장례식장으로 납품을 완료할 때까지의 사이클 통합 관리.

## 2. 주요 개념 및 구현 내역

### 2.1 통합 외주업체 계정 (`business_type`: 'all')

* **단일 업체 다수 카테고리 지원:** 기존에는 "꽃집", "납골당", "용품"으로 분리되어 있던 카테고리에 **종합('all')** 과 **근조화환('wreaths')** 타입을 새로 추가.
* **적용 화면:**
  * 가입 폼 (`PartnerApply.jsx`): 종합(장례용품/입관꽃/화환 통합) 가입 옵션 제공.
  * 발주서 작성 모달 (`OrderModal.jsx`): 팀장이 업체 리스트를 조회할 때 '종합' 카테고리가 뱃지와 함께 최상단에 표출됨. یک 장바구니에 3가지 타 카테고리 상품 동시 발주 가능.

### 2.2 기사용 노로그인 납품 시스템 (`DriverDelivery.jsx`)

* **목적:** 외주업체에서 고용한 외부 배송기사 혹은 퀵서비스 기사가 복잡한 회원가입이나 앱 설치 없이 알림 링크만으로 배송 완료 보고를 할 수 있도록 지원.
* **프로세스 동선:**
  1. 외주업체 대시보드 (`VendorDashboard.jsx`)에서 [기사 전용링크 복사] 클릭.
  2. 기사에게 카카오톡/문자 발송.
  3. 기사가 링크 클릭 시 `/delivery/:orderId` 로 접속 (모바일 최적화 UI).
  4. 도착지 위치, 고인명, 입관일시, 담당 팀장 연락처 및 발주 목록 확인.
  5. 기사가 현장 납품 사진 촬영(업로드) 후 시스템 내 '본인 이름'을 선택하여 [납품완료] 처리.
  
### 2.3 양방향 동시 알림 시스템 (`notifications` DB 기반)

* 기사가 [납품완료] 버튼을 클릭하는 즉시, Supabase `deliveries`, `orders` 테이블 trạng 태 변경 후 동시에 다음 2곳으로 노티피케이션 데이터 INSERT.
  1. **담당 팀장 (`team_leader_id`)**: "납품 완료 알림 🚚 ... 배송기사 OOO"
  2. **해당 외주업체 사장님 (`vendor_id` -> `user_id`)**: "배송기사 OOO가 납품 완료 처리를 확인 ✅"

## 3. 핵심 파일 요약

* **`src/pages/VendorDashboard.jsx`**: 외주업체의 기본 대시보드. 링크 복사 버튼 추가 및 입관/발인일시 추가 렌더링.
* **`src/components/teamleader/OrderModal.jsx`**: 팀장이 발주하는 메인 UI 모달. 특이사항(입관시간 등) 자동 프리필.
* **`src/pages/DriverDelivery.jsx`**: (신규) 기사용 모바일 전면 노로그인 배송 링크 처리 페이지.
* **`seed_vendor_test.sql`**: '하늘꽃 종합상사' 통합 테스트 계정 및 3종 제품 풀세팅 구문 최신화 버전 유지.
