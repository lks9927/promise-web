# 2단계: 데이터 병렬 호출 / 캐싱

> 난이도: 🟢 쉬움 | 소요: 반나절 | 효과: 체감 속도 개선

## 현재 문제
- `fetchData()` 안에서 6개 테이블을 **순차 호출** (하나 끝나야 다음 시작)
- 각 호출 0.3~0.5초 × 6개 = 약 2~3초 대기

## 목표
- `Promise.all`로 **동시 호출** → 가장 느린 호출 1개 시간만 소요 (0.5초 이내)
- 탭 전환 시 이미 가져온 데이터는 재호출하지 않음

## AI 지시

```
AdminDashboard.jsx의 fetchData 함수를 최적화해줘.

변경 내용:
1. 현재 순차 호출(await 6번)을 Promise.all로 병렬 호출로 변경
2. 예시:
   const [settlements, cases, partners, configs, coupons, vendors] = await Promise.all([
     supabase.from('settlements').select(...),
     supabase.from('funeral_cases').select(...),
     supabase.from('partners').select(...),
     supabase.from('system_config').select(...),
     supabase.from('coupons').select(...),
     supabase.from('vendors').select(...)
   ]);
3. 에러 처리: 한 테이블 실패해도 나머지는 정상 반영되게 Promise.allSettled 사용 고려
4. TeamLeaderDashboard, DealerDashboard, MasterDashboard도 같은 방식 적용

주의: 4단계에서 추가한 Realtime 구독 코드는 건드리지 마
```
