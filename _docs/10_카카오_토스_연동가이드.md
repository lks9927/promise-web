# 📱 알리고(Aligo) 문자 & 💸 토스페이먼츠 연동 가이드

본 문서는 `10년의 약속` 웹 서비스에 알리고(Aligo) SMS/LMS 알림 시스템과 토스페이먼츠(Toss Payments) 가상결제 시스템을 도입하기 위한 **준비 사항 및 작동 시나리오**를 정리한 문서입니다.

---

## 1. 📱 알리고(Aligo) 일반 문자(SMS/LMS) 연동 준비

카카오 비즈니스 채널 심사 지연 및 반려 리스크를 회피하고, 까다로운 템플릿 사전 승인 없이 즉시 발송 가능한 **알리고(Aligo)**를 통해 일반 문자로 시스템을 구축합니다.

### ✅ 대표님 준비 사항 (발급 및 세팅)
1. **알리고 회원가입 및 충전**: [알리고(aligo.in)](https://smartsms.aligo.in/) 가입 후 최소 금액 충전
2. **발신번호 등록**: 사이트 내 [발신번호 관리]에서 문자 발송 시 표시될 번호 사전 인증 완료
3. **API Key 발급**: 상단 메뉴 [신청/조회] -> [API 연동신청]에서 발급
4. **접속 정보 전달**: 
   * `알리고 아이디`
   * `API 인증키(Auth Key)`
   * `등록한 발신번호 (예: 01012345678)`

### 💻 개발자(AI) 코딩 계획 (진행 중)
* 고객 접속 폼(랜딩페이지)에서 접수가 완료되거나 마스터/팀장 간 배정 상태가 변경될 때마다 Supabase DB 트리거 이벤트가 발생합니다.
* 예약된 `aligo-sms` Supabase Edge Function이 즉각적으로 트리거를 가로채어 **자동으로 고객/팀장/마스터에게 맞춤형 일반 문자가 발송**됩니다.
* 월요일에 발급받으신 정보 3가지를 서버 환경변수에 입력만 하시면 즉시 구동됩니다.

---

## 2. 💸 토스페이먼츠 (결제 시스템) 연동 준비

고객 장례 대금이나 파트너 수수료를 간편하게 웹/모바일에서 결제받기 위해 사용합니다. 신용카드, 계좌이체, 카카오페이 등 모든 간편결제를 한 번에 지원합니다.

### ✅ 대표님 준비 사항 (가맹점 등록)
1. **토스페이먼츠 가입**: [토스페이먼츠 개발자센터](https://developers.tosspayments.com/) 회원가입
2. **기본 키 세팅 (테스트 모드 우선)**: 가입 직후 발급되는 **[테스트용]** `Client Key`와 `Secret Key`를 먼저 개발자(AI)에게 전달
   * *이 키로 먼저 코딩과 테스트 장례 결제를 완벽하게 시뮬레이션할 수 있습니다.*
3. **실 가맹점 심사 신청**: 사업자등록증, 통장 사본, 서비스 운영 사이트(현재 배포된 Vercel 도메인)를 제출하여 카드사 심사 신청 (약 1~2주 소요)
4. **라이브 키 교체**: 심사 통과 후 발급되는 **[실제용]** 키를 전달해주시면 코드를 교체하여 실제 돈이 오가도록 변경합니다.

### 💻 개발자(AI) 코딩 계획 (2가지 유력 시나리오)

#### 시나리오 A: [상주/고객] 장례 대금 간편 결제
1. **결제 링크 발송**: 장례가 종료되면 관리자(팀장)가 대시보드에서 '고객에게 결제 요청' 버튼 클릭
2. **알림톡 도착**: 고객 카카오톡으로 **[결제창 열기]** 링크가 발송됨
3. **결제 진행**: 브라우저가 열리면서 토스페이먼츠 결제창 노출 -> 카드 할부 및 카카오페이 1초 결제 완료
4. **자동 정산**: DB에 결제 완료가 즉시 찍히며, 배정된 팀장의 정산(수수료 차감 후 지급액)이 대시보드에 업데이트됨

#### 시나리오 B: [마스터/파트너] 예치금 및 본사 수수료 결제
1. **수수료 청구**: 마스터가 '식구(딜러)'를 등록하여 본사 승인을 받을 때, 관리비/수수료 결제 창이 뜸
2. **법인/개인 카드 결제**: 마스터가 토스페이먼츠 창을 통해 본사로 수수료 지불
3. **권한 자동 승인**: 결제가 완료되면 대기 중이던 식구(딜러)가 즉시 `승인(approved)` 상태로 전환되어 활동 가능
