## 1. 유저 구조 & 금전 흐름

### 📌 전체 유저 역할

| 역할 | 설명 | 금전 관계 |
|---|---|---|
| **전체 관리자** | 모든 관리 총괄 (본사) | 수수료 수령 / 지급금 집행 |
| **마스터 팀장** | 하위 일반팀장 관리 | 본사에 장례수수료 **입금** / 일반팀장에게 관리비 **지급** |
| **팀장** | 장례 입찰·진행 | 본사에 장례수수료 **입금** |
| ── 경계선 ── | *아래 유저는 장례진행상황만 열람 가능* | |
| **마스터 딜러** | 지역별 팀장·딜러 관리 | 장례 실행 시 20만원 **수령** / 일반딜러에게 관리비 **지급** |
| **딜러** | 일반회원 영업 | 장례 실행 시 20만원 **수령** |
| **고객** | 장례 신청 | 장례 실행 시 10만원 **수령** |

### 💵 입금 흐름 (본사로 들어오는 돈)

```
마스터 팀장 ──→ 본사  (장례수수료, 현금 입금)
팀장       ──→ 본사  (장례수수료, 현금 입금)
```

- **결제 방식**: 무조건 **현금 입금**
- **증빙 발행**: 세금계산서 또는 현금영수증

### 💸 지급 흐름 (본사에서 나가는 돈)

```
본사 ──→ 마스터 딜러  (장례 실행 시 20만원)
본사 ──→ 딜러         (장례 실행 시 20만원)
본사 ──→ 고객         (장례 실행 시 10만원)
```

### 🔒 데이터 접근 경계

```
┌─────────────────────────────────────────────┐
│  전체 관리자 / 마스터 팀장 / 팀장            │
│  → 전체 데이터 접근 가능                     │
├─────────────────────────────────────────────┤
│  마스터 딜러 / 딜러 / 고객                   │
│  → 장례 진행상황만 열람 가능                  │
└─────────────────────────────────────────────┘

※ 고객 중 본사 직접 유입 건 → 본사 + 해당 담당 팀장만 열람 가능
```

## 2. 정산 방식 (현재)

### ✅ 현재 운영 방식: 세무사 수동 정산

| 항목 | 내용 |
|---|---|
| **입금 확인** | 본사 계좌로 현금 입금 → 세무사가 확인 |
| **증빙 발행** | 세금계산서 또는 현금영수증 발행 (세무사 처리) |
| **지급 처리** | 마스터딜러/딜러/고객 → 세무사가 원천징수 후 계좌이체 |
| **정산 주기** | 세무사와 협의하여 결정 |

### 💻 프로그램에서 필요한 기능 (현재)

- [ ] 장례 완료 시 **정산 내역 자동 생성** (누구에게 얼마 지급해야 하는지)
- [ ] 관리자 대시보드에서 **정산 대기 목록** 확인
- [ ] 정산 완료 처리 (세무사가 입금 후 관리자가 '지급 완료' 체크)
- [ ] 정산 이력 조회 및 리포트

## 3. 토스페이먼츠 Payouts (추후 도입 예정)

> ⚠️ **현재 미도입** — 월 기본료 30만원 발생으로 사업 안정화 후 전환 예정

### 🔮 추후 전환 시 변경점

| 현재 (세무사) | 추후 (토스 Payouts) |
|---|---|
| 세무사가 수동 계좌이체 | API로 자동 송금 |
| 정산 처리 1~3일 소요 | 즉시 또는 예약 송금 |
| 수동 원천징수 계산 | 시스템에서 3.3% 자동 공제 후 송금 |

### 📝 토스 도입 시 필요한 준비 (메모)

1. 토스페이먼츠 가맹점 가입 (월 기본료 30만원)
2. Payouts API 보안키 발급 (JWE 암호화 필수)
3. 셀러(서브몰) 등록 → 마스터딜러/딜러/고객 계좌 등록
4. Supabase Edge Function에서 API 호출 (Secret Key 보호)
5. 기존 `src/lib/tossPayouts.js` 코드 활성화 및 연동
