# 🧾 10년의 약속 - 세금 관리 시스템 명세서

> **이 문서는 세금 계산, 원천징수 관리, 부가세 관리, 종합소득세 예측 기능을 정의합니다.**
> **모든 AI 및 개발자는 관련 코드 수정 전 반드시 이 문서를 먼저 읽어야 합니다.**

## 1. 세금 유형 정리

### 본사가 다루는 세금 3가지

| 세금 | 대상 | 세율 | 처리 방식 | 납부 주기 |
|------|------|:----:|----------|----------|
| **원천징수** | 딜러 커미션 지급 시 | 3.3% | 커미션에서 **차감** 후 송금 | 반기 (7월, 1월) |
| **부가세 (VAT)** | 팀장 수수료 입금 시 | 10% | 수수료에 **가산**하여 수령 | 분기 (1/4/7/10월) |
| **종합소득세** | 본사 연간 소득 | 누진세율 | 연 1회 신고 | 5월 |

---

## 2. 원천징수 (3.3%) - 딜러 커미션

### 흐름
```
딜러 커미션 200,000원 발생
       ↓
원천징수 3.3% = 6,600원 차감
       ↓
딜러에게 실지급: 193,400원
세금 통장에 보관: 6,600원 ← 모아서 반기 납부
```

### DB 저장 구조 (현재 코드)
```sql
-- apply_settlement_final.sql 에서 자동 계산
INSERT INTO settlements (
    case_id, recipient_id, amount, type, status,
    tax_type, base_amount, tax_amount, net_amount
) VALUES (
    NEW.id, v_dealer_id,
    v_net_amt,           -- 193,400 (실지급액)
    'dealer_commission', 'pending',
    'withholding_33',    -- 세금 유형
    v_dealer_comm,       -- 200,000 (원래 금액)
    v_tax_amt,           -- 6,600 (원천징수액)
    v_net_amt            -- 193,400 (실지급액)
);
```

### 원천징수 적용 대상

| 정산 유형 | 세금 유형 | 적용 |
|----------|----------|:----:|
| `dealer_commission` (딜러 커미션) | `withholding_33` | ✅ 3.3% 차감 |
| `dealer_override` (마스터딜러 관리커미션) | `withholding_33` | ✅ 3.3% 차감 |
| `leader_override` (마스터팀장 관리커미션) | `none` | ❌ 사업자 → 원천징수 없음 |
| `customer_cashback` (고객 캐시백) | `none` | ❌ 세금 없음 |

### 납부 일정
| 귀속 기간 | 납부 기한 | 비고 |
|----------|----------|------|
| 1월 ~ 6월 | 7월 10일 | 상반기 |
| 7월 ~ 12월 | 다음해 1월 10일 | 하반기 |

## 3. 부가세 (10%) - 팀장 수수료

### 흐름
```
팀장 수수료 700,000원 발생
       ↓
부가세 10% = 70,000원 가산
       ↓
팀장이 본사에 입금: 770,000원
세금 통장에 보관: 70,000원 ← 모아서 분기 납부
```

### DB 저장 구조 (현재 코드)
```sql
-- apply_settlement_final.sql 에서 자동 계산
INSERT INTO settlements (
    case_id, recipient_id, amount, type, status,
    tax_type, base_amount, tax_amount, net_amount
) VALUES (
    NEW.id, v_team_leader_id,
    v_net_amt,                -- 770,000 (총 입금액)
    'usage_fee_remittance', 'pending',
    'vat_10',                 -- 세금 유형
    NEW.usage_fee,            -- 700,000 (원래 금액)
    v_tax_amt,                -- 70,000 (부가세)
    v_net_amt                 -- 770,000 (총 입금액)
);
```

### 납부 일정
| 과세 기간 | 납부 기한 | 비고 |
|----------|----------|------|
| 1월 ~ 3월 | 4월 25일 | 1기 예정 |
| 4월 ~ 6월 | 7월 25일 | 1기 확정 |
| 7월 ~ 9월 | 10월 25일 | 2기 예정 |
| 10월 ~ 12월 | 다음해 1월 25일 | 2기 확정 |

## 4. 종합소득세 예측

### 계산 공식
```
연간 수입(+)
  - 수수료 입금 총액 (usage_fee_remittance 합계)
  
연간 지출(-)
  - 커미션 지급 총액 (dealer_commission + dealer_override + leader_override + customer_cashback 합계)
  - 회사 운영 경비 (수동 입력)

= 과세 소득

→ 누진세율 적용 → 예상 종합소득세
```

### 종합소득세 누진세율 (2024 기준)

| 과세표준 | 세율 | 누진공제 |
|---------|:----:|:-------:|
| 1,400만원 이하 | 6% | 0원 |
| 1,400 ~ 5,000만원 | 15% | 126만원 |
| 5,000 ~ 8,800만원 | 24% | 576만원 |
| 8,800 ~ 1.5억원 | 35% | 1,544만원 |
| 1.5억 ~ 3억원 | 38% | 1,994만원 |
| 3억 ~ 5억원 | 40% | 2,594만원 |
| 5억 ~ 10억원 | 42% | 3,594만원 |
| 10억원 초과 | 45% | 6,594만원 |

### 종소세 예측에 필요한 데이터

| 데이터 | 출처 | 상태 |
|--------|------|:----:|
| 수수료 입금 내역 | `settlements` 테이블 자동 기록 | ✅ 있음 |
| 커미션 지급 내역 | `settlements` 테이블 자동 기록 | ✅ 있음 |
| 원천징수 누적액 | `settlements.tax_amount` 합계 | ✅ 있음 |
| 부가세 누적액 | `settlements.tax_amount` 합계 | ✅ 있음 |
| 회사 운영 경비 | **수동 입력 필요** | ❌ 없음 |

## 8. 필요한 DB 테이블 (신규)

### company_expenses (회사 경비)
```sql
CREATE TABLE company_expenses (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    date DATE NOT NULL,
    category TEXT NOT NULL,           -- 임대료, 통신비, 차량 등
    amount INTEGER NOT NULL,          -- 금액
    memo TEXT,                        -- 메모
    receipt_url TEXT,                  -- 증빙 파일 URL
    created_at TIMESTAMPTZ DEFAULT now(),
    created_by UUID REFERENCES profiles(id)   -- 입력자
);
```
